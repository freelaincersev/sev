## 5. 측정 가능한 성공 기준

| 기준 | 합격 조건 | 현재 |
|---|---|---|
| 자료 처리 | paste/MD 자료 1건 추가 시 status=`ready` + chunk·embedding row가 DB에 생성 | ❌ 미구현 |
| 검색 정확도 | 질문 시 관련 chunk가 project 범위 내에서 top-k 반환 (교차 유저/프로젝트 0) | RPC는 존재, 앱 연결 ❌ |
| 인용 | 답변에 최소 1개 citation(원문 snippet + 출처) 표시 | ❌ 미구현 |
| Context Packet | packet 생성 후 "복사" 버튼으로 LLM-ready 텍스트가 클립보드에 복사 | ❌ 미구현 |
| E2E 무오류 | 신규 유저→프로젝트→자료→packet까지 콘솔/서버 에러 0 | 부분(프로젝트까지만 확인) |
| 제품 지표 (전략 §10.5) | Weekly Context Packets Created, Packets Copied to External LLM | **미측정** (실사용자 0) |

## 6. 가장 큰 리스크 1개

| 리스크 | 설명 | 완화책 |
|---|---|---|
| **가치 전달 핵심 루프가 통째로 미구현** | 현재 존재하는 건 인증 + 프로젝트 CRUD뿐 — 어떤 스타터와도 구분 안 되는 껍데기. 전략 §8은 "품질은 ingestion·chunking·retrieval에서 갈린다"고 명시하는데, 바로 그 어렵고 차별적인 부분(ingestion→retrieval→Context Packet)이 하나도 없어 **핵심 가설이 전혀 검증되지 않음**. | 향후 4주를 오직 "자료 1건 → 인용된 답변 → packet 1개"(M2–M4) 완성에만 집중하고, 그 외(연동·결제·PDF·팀)는 전부 뒤로 미룸. |
