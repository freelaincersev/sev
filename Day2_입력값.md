## 2. 고객 가설 3개 (누가 · 어떤 상황에서 · 왜)

근거: 전략문서 §3(핵심 고객), §3.7(초기 사용자의 하루), §2.2(실제 불편).

1. **누가** AI 네이티브 솔로 창업자·빌더 / **어떤 상황** 하루에 ChatGPT·Claude·Cursor를 오가며 같은 프로젝트 배경을 반복 설명·재업로드 / **왜** 도구 간 맥락이 공유되지 않아 매번 자신을 다시 설명해야 함. (§2.1, §3.7)
2. **누가** 리서치·자료가 계속 쌓이는 1인 지식노동자 / **어떤 상황** PDF·메모·고객 인터뷰는 많은데 AI가 바로 쓸 형태가 아님 / **왜** 자료가 AI-ready(Markdown·검색 가능)로 정리돼 있지 않음. (§2.2 문제1)
3. **누가** 민감 자료(사업 아이디어·계약·재무)를 다루는 솔로 오퍼레이터 / **어떤 상황** 전체 자료를 AI에 통째로 올리기는 불안 / **왜** 필요한 최소 맥락만 골라 전달하고 싶음. (§2.2 문제3, §7)

> 미확인(Day02 조사): 실제 이 세그먼트의 규모, 반복 설명에 쓰는 시간, 현재 대체 방법.

## 4. 차별화 가설

근거: 전략문서 §11.5 wedge, §11.9.

경쟁사는 자료를 **자기 워크스페이스/모델 안에 가두거나**(Notion AI, ChatGPT/Claude Projects) **검색까지만**(Dropbox Dash) 제공한다. Sev의 가설은 —

> 자료를 **model-agnostic Context Packet**으로 조립해 **어떤 LLM에도 이동**시키고(portability), 답변에 **citation**을 붙이며, 사용자가 **소유·export** 가능한 Markdown 기억으로 둔다. 이 "이동성 + 신뢰(citation) + 소유권"이 기존 도구에 없는 wedge다. ("LLMs change. Your context stays.")

**검증 포인트(Day02):** 이 차별점이 실제 구매 이유가 되는지 — 즉 "특정 LLM에 갇히는 불편"과 "매번 재설명하는 피로"가 지불 의사로 이어지는지.
